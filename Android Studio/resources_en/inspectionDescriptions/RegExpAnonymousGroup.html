Докладывает об анонимных захватываемых группах и обратных числовых ссылках в регулярных выражениях. These are only reported when the RegExp dialect supports named group and named group references. Named groups and named back references improve code readability and are recommended to use instead. When a capture is not needed, matching can be more performant and use less memory by using a non-capturing group, i.e. (?:xxx) instead of (xxx).

New in 2017.2
